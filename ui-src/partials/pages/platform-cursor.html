接入流程

按顺序完成，失败项会直接显示原因。

1
准备代理模型

至少启用一个 OpenAI 兼容模型，Cursor 模型选择器会读取这里的路由。

2
写入 Cursor 状态

备份 Cursor 原状态后写入 BYOK 会话、默认模型和代理设置。

3
启动本机代理

保持 AnyBridge 后台服务运行，Cursor 的请求会进入本机 MITM 入口。

4
重启 Cursor

重启后 Cursor 才会重新读取 settings.json 和 state.vscdb。

隐私边界

AnyBridge 方案在本机代理内构造请求，不把你的 provider key 交给 Cursor 后端；但 Cursor 客户端仍会把编辑器上下文发到本机代理处理。

排障入口

遇到模型不显示、证书报错、Cursor 无响应时，先跑环境检测，再查看代理日志和 turn artifacts。
