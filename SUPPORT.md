# Support

Use GitHub Issues for reproducible bugs, feature requests, and documentation gaps.

Before opening an issue:

- Search existing issues.
- Check `README.md` and `docs/`.
- Remove API keys, tokens, local paths, emails, and captured payloads from logs.
- Include OS, AnyBridge version, target tool version, and provider type.

Security reports should follow [SECURITY.md](SECURITY.md).
