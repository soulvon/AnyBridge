    接入流程

    按顺序完成，失败项会直接显示原因。

    1
    准备代理模型

    至少启用一个 OpenAI 兼容模型，Cursor 模型选择器会读取这里的路由。

    2
    写入 Cursor 状态

    备份 Cursor 原状态后写入 BYOK 会话、默认模型和代理设置。

    3
    启动本机代理

    保持 AnyBridge 后台服务运行，Cursor 的请求会进入本机 MITM 入口。

    4
    重启 Cursor

    重启后 Cursor 才会重新读取 settings.json 和 state.vscdb。

    项目页面 查看源码和说明
    如果安装失败，AnyBridge 会显示 GitHub 下载或 IDE 安装命令的原始错误，方便继续排查。
    需要重启 IDE

    代理已启动，需重启 IDE 才能生效。

    Quick Start
    
          
    把模型接入工作流，只需要四步
    
          
    按顺序配置供应商、模型映射、本地代理和平台接入。AnyBridge 会尽量保留原始配置，并把关键错误暴露出来方便排查。
    
        
    
        
      
    
      
    
        
    
          01
          添加供应商
          
    填入 API Host、Key 和默认模型，确认协议类型能连通。
    
        
    
        
    
          02
          建立模型映射
          
    把 IDE 槽位映射到真实供应商模型，可按顺序添加备用目标。
    
        
    
        
    
          03
          启动本地代理
          
    统一暴露 OpenAI/Anthropic 兼容入口，并启用重试和日志。
    
        
    
        
    
          04
          接入平台
          
    为 Windsurf、Codex、Cursor 等平台写入配置，重启后生效。
